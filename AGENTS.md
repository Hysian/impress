# Repository Guidelines

## Project Structure & Module Organization
- `src/` contains the React + TypeScript app. Key areas: `src/components/` (shared UI), `src/pages/` (route-level screens), `src/router/` (routing setup), `src/i18n/` (localization), `src/index.css` (global styles).
- `public/` holds static assets copied as-is.
- `index.html` is the Vite entry point.
- `out/` is the Vite build output directory (generated; do not edit).
- `auto-imports.d.ts` is generated by unplugin-auto-import (do not edit manually).
- `nginx.conf` contains deployment server configuration.

## Build, Test, and Development Commands
Use `pnpm` (see `pnpm-lock.yaml`).

```bash
pnpm install        # install dependencies
pnpm dev            # start Vite dev server (http://localhost:3000)
pnpm build          # build for production into out/
pnpm preview        # serve the production build locally
pnpm lint           # run ESLint on src/
pnpm type-check     # run TypeScript type checking
```

## Coding Style & Naming Conventions
- TypeScript + React with functional components in `*.tsx`.
- Indentation: 2 spaces; quotes: double; semicolons present (follow existing files).
- ESLint is configured in `eslint.config.ts`; fix lint issues before committing.
- Path alias `@` maps to `src` (see `vite.config.ts`).
- Tailwind CSS is enabled; prefer utility classes for component styling.
- React hooks and router helpers are auto-imported; avoid redundant imports when possible.

## Testing Guidelines
- No test framework or test files are currently configured in this repo.
- If adding tests, introduce a runner (e.g., Vitest) and keep names consistent (e.g., `*.test.tsx`) under `src/`.

## Commit & Pull Request Guidelines
- Git history is not available in this checkout, so no existing commit convention could be derived.
- Recommended commit style: short, imperative subject line (e.g., `Add pricing page CTA`).
- PRs should include: a concise summary, linked issues (if any), and screenshots for UI changes, plus the commands run (e.g., `pnpm lint`, `pnpm type-check`).

## Configuration Notes
- Vite build/runtime flags: `BASE_PATH`, `IS_PREVIEW`, `PROJECT_ID`, `VERSION_ID`, `READDY_AI_DOMAIN` (see `vite.config.ts`).
